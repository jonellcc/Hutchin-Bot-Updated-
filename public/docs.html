Command Structure

    name: "Name of command",
    info: "description of command",

        // Command logic here
    }

Event Handlers

Hutchins Bot 2.0 provides several event handlers for different interaction types:

onLaunch

Executed when the command is called. Handles initialization tasks and responds to the event.

Core Function

onEvents with target

Triggered with arguments including the target for more complex command handling.

Advanced

onReply

Executed when a user replies to a specific message for context-sensitive responses.

Interactive

callReact

Invoked when a user reacts to a message, enabling confirmation actions.

Reaction-based

noPrefix

Allows command execution without a prefix for natural interactions.

Special

Actions

The actions object provides methods for responding to user interactions:

Reply Message

actions.reply("Hello!");

Respond to a user directly.

Send Message

actions.send("Hello, everyone!");

Send a message to the thread.

React to Message

actions.react("üî•");

React to the current message with an emoji.

Edit Message

const loading = await actions.reply("Loading...");
actions.edit("Hello", loading.messageID);

Edit a previously sent message.

Kick User

actions.kick(userID);

Remove a user from the group.

Leave Group

actions.leave();

Remove the bot from the current group.

Share Contact

actions.share(contact, senderID);

Share a contact with a specific user.

Advanced Features

Nickname (Aliases)

You can use aliases for commands by defining a nickName property:

module.exports = {
    name: "test",
    nickName: ["test", "testing"],
    onLaunch: async function ({ api, event, actions }) {
        await actions.reply("This is a test command.");
    }
};

dmUser Property

The dmUser property indicates whether a command can be executed through direct messages:

module.exports = {
    name: "example",
    dmUser: true,
    onLaunch: async function ({ api, event, actions }) {
        await actions.reply("This command can be executed in DMs.");
    }
};

onLaunch Function

The onLaunch function is executed when the command is called. It can handle initialization tasks and respond to the event.

onLaunch: async function ({ api, event, actions }) {
    const message = "Command executed!";
    await actions.reply(message);
}

Parameters

  • api: The bot API interface
  • event: The triggering event object
  • actions: Object containing response methods

onEvents with target

The onEvents function is triggered with arguments, including the target for more complex command handling.

onEvents: async function ({ api, event, target }) {
    const targetText = target.join(" ");
    if (!targetText) return actions.reply("Provide the Text");
    // Function logic here 
}

Parameters

  • api: The bot API interface
  • event: The triggering event object
  • target: Array of target arguments

onReply Function

The onReply function is executed when a user replies to a specific message. This allows for context-sensitive responses.

onReply: async function({ reply, api, event }) {
    const response = `Your reply is: ${reply}`;
    await api.sendMessage(response, event.threadID);
}

Parameters

  • reply: The reply content from user
  • api: The bot API interface
  • event: The triggering event object
  • global.client.onReply: is used to track and manage replies to specific messages. It stores the reply context (like messageID and author), allowing the bot to match replies with the correct message and respond accordingly. It also handles auto-cleanup of expired or processed replies.

callReact Function

The callReact function is invoked when a user reacts to a message. It enables confirmation actions or handling reactions.

callReact: async function ({ reaction, event, api }) {
    if (reaction === 'üëç') {
        await actions.reply("Confirmed!");
    } else if (reaction === 'üëé') {
        await actions.reply("Cancelled.");
    }
}

Parameters

  • reaction: The reaction emoji used
  • api: The bot API interface
  • event: The triggering event object

noPrefix Function

The noPrefix function allows the command to be executed without a prefix, useful for more natural interactions.

noPrefix: async function ({ api, event }) {
    await actions.reply("This command can be executed without a prefix.");
}

Parameters

  • api: The bot API interface
  • event: The triggering event object
